# CMS для Сергея — прогресс

Цель: Сергей сам редактирует любое фото галереи, услуги/цены, условия (правила проживания), телефон — через веб-панель `/admin`, без участия разработчика. Технология — Decap CMS поверх GitHub+Vercel (правка → коммит → автопересборка).

## Этап 1 — контент в редактируемый формат (в процессе)
- [ ] Перенести фото галереи из `src/assets/images` (вшиты в билд) в `public/images/gallery` (обычные файлы, CMS сможет их заменять)
- [ ] `content/gallery.json` — список фото вместо `galleryPhotos` в data.ts
- [ ] `content/services.json` — услуги/цены вместо `amenitiesList`
- [ ] `content/rules.json` — правила проживания вместо хардкода в `HouseRules.tsx`
- [ ] `content/settings.json` — телефон (сейчас захардкожен в 3 местах: Header, Footer, FinalCTA)
- [ ] Обновить компоненты, чтобы читали из этих файлов
- [ ] Собрать, проверить визуально (Playwright), закоммитить

## Этап 2 — сама CMS
- [ ] `public/admin/index.html` + `config.yml` (описание полей для Сергея)
- [ ] Серверные функции для входа через GitHub (`api/auth.js`, `api/callback.js`)
- [ ] **Нужно от Димы:** создать GitHub OAuth App (дам точные значения когда дойдём до этого шага), передать Client ID/Secret
- [ ] Прописать переменные окружения в Vercel
- [ ] Проверить цикл целиком: вход → правка → коммит → сайт обновился

## Решения
- Reviews (отзывы) и FAQ пока НЕ переношу в CMS — Дима просил фото/услуги/условия/телефон, не всё подряд. Можно добавить позже отдельным шагом.
